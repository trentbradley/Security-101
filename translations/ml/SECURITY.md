## സുരക്ഷ

മൈക്രോസോഫ്റ്റ് ഞങ്ങളുടെ സോഫ്റ്റ്വെയർ ഉൽപ്പന്നങ്ങളും സേവനങ്ങളും, അതിൽ ഉൾപ്പെടുന്ന എല്ലാ സോഴ്‌സ് കോഡ് റിപോസിറ്ററികളും, ഗൗരവത്തോടെ കൈകാര്യം ചെയ്യുന്നു. ഇതിൽ [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet), [Xamarin](https://github.com/xamarin) എന്നിവയുടെ GitHub ഓർഗനൈസേഷനുകൾ ഉൾപ്പെടുന്നു.

നിങ്ങൾക്ക് മൈക്രോസോഫ്റ്റ് ഉടമസ്ഥതയിലുള്ള ഏതെങ്കിലും റിപോസിറ്ററിയിൽ [മൈക്രോസോഫ്റ്റിന്റെ സുരക്ഷാ ദൗർബല്യത്തിന്റെ നിർവചനം](https://aka.ms/security.md/definition) പാലിക്കുന്ന ഒരു സുരക്ഷാ ദൗർബല്യം കണ്ടെത്തിയതായി തോന്നിയാൽ, ദയവായി താഴെ വിവരിച്ചിരിക്കുന്നതുപോലെ ഞങ്ങളോട് റിപ്പോർട്ട് ചെയ്യുക.

## സുരക്ഷാ പ്രശ്നങ്ങൾ റിപ്പോർട്ട് ചെയ്യൽ

**ദയവായി പൊതുവായ GitHub ഇഷ്യൂസുകൾ വഴി സുരക്ഷാ ദൗർബല്യങ്ങൾ റിപ്പോർട്ട് ചെയ്യരുത്.**

പകരം, ദയവായി മൈക്രോസോഫ്റ്റ് സെക്യൂരിറ്റി റെസ്പോൺസ് സെന്ററിലേക്ക് (MSRC) [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report) എന്ന ലിങ്ക് വഴി റിപ്പോർട്ട് ചെയ്യുക.

ലോഗിൻ ചെയ്യാതെ സമർപ്പിക്കാൻ നിങ്ങൾക്ക് മുൻഗണനയുണ്ടെങ്കിൽ, [secure@microsoft.com](mailto:secure@microsoft.com) എന്ന ഇമെയിലിലേക്ക് അയയ്ക്കുക. നിങ്ങളുടെ സന്ദേശം ഞങ്ങളുടെ PGP കീ ഉപയോഗിച്ച് എൻക്രിപ്റ്റ് ചെയ്യാൻ കഴിയും; ദയവായി അത് [Microsoft Security Response Center PGP Key പേജ്](https://aka.ms/security.md/msrc/pgp) നിന്ന് ഡൗൺലോഡ് ചെയ്യുക.

24 മണിക്കൂറിനുള്ളിൽ നിങ്ങൾക്ക് ഒരു പ്രതികരണം ലഭിക്കണം. എന്തെങ്കിലും കാരണത്താൽ നിങ്ങൾക്ക് പ്രതികരണം ലഭിക്കാത്ത പക്ഷം, നിങ്ങളുടെ ആദ്യ സന്ദേശം ഞങ്ങൾ സ്വീകരിച്ചിട്ടുണ്ടോ എന്ന് ഉറപ്പാക്കാൻ ഇമെയിൽ വഴി പിന്തുടരുക. കൂടുതൽ വിവരങ്ങൾ [microsoft.com/msrc](https://www.microsoft.com/msrc) ൽ ലഭ്യമാണ്.

ദയവായി താഴെപ്പറയുന്ന വിവരങ്ങൾ (നിങ്ങൾക്ക് നൽകാൻ കഴിയുന്നത്ര) ഉൾപ്പെടുത്തുക, ഇതിലൂടെ പ്രശ്നത്തിന്റെ സ്വഭാവവും പരിധിയും ഞങ്ങൾക്കു മനസ്സിലാക്കാൻ സഹായിക്കും:

  * പ്രശ്നത്തിന്റെ തരം (ഉദാ: ബഫർ ഓവർഫ്ലോ, SQL ഇൻജക്ഷൻ, ക്രോസ്-സൈറ്റ് സ്ക്രിപ്റ്റിംഗ്, മുതലായവ)
  * പ്രശ്നം പ്രകടമാകുന്ന സോഴ്‌സ് ഫയലുകളുടെ പൂർണ്ണ പാതകൾ
  * ബാധിതമായ സോഴ്‌സ് കോഡിന്റെ സ്ഥാനം (ടാഗ്/ബ്രാഞ്ച്/കമിറ്റ് അല്ലെങ്കിൽ നേരിട്ടുള്ള URL)
  * പ്രശ്നം പുനരുത്പാദിപ്പിക്കാൻ ആവശ്യമായ പ്രത്യേക കോൺഫിഗറേഷൻ
  * പ്രശ്നം പുനരുത്പാദിപ്പിക്കാൻ സ്റ്റെപ്പ്-ബൈ-സ്റ്റെപ്പ് നിർദ്ദേശങ്ങൾ
  * പ്രൂഫ്-ഓഫ്-കോൺസെപ്റ്റ് അല്ലെങ്കിൽ എക്സ്പ്ലോയിറ്റ് കോഡ് (സാധ്യമായെങ്കിൽ)
  * പ്രശ്നത്തിന്റെ സ്വാധീനം, ഒരു ആക്രമി പ്രശ്നം എങ്ങനെ ഉപയോഗപ്പെടുത്തും എന്നതടക്കമുള്ളവ

ഈ വിവരങ്ങൾ നിങ്ങളുടെ റിപ്പോർട്ട് വേഗത്തിൽ പരിഗണിക്കാൻ ഞങ്ങളെ സഹായിക്കും.

നിങ്ങൾ ഒരു ബഗ് ബൗണ്ടിക്കായി റിപ്പോർട്ട് ചെയ്യുകയാണെങ്കിൽ, കൂടുതൽ സമ്പൂർണ്ണമായ റിപ്പോർട്ടുകൾ ഉയർന്ന ബൗണ്ടി അവാർഡിലേക്ക് സംഭാവന ചെയ്യാൻ കഴിയും. ഞങ്ങളുടെ സജീവ പ്രോഗ്രാമുകളെക്കുറിച്ചുള്ള കൂടുതൽ വിവരങ്ങൾക്ക് [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) പേജ് സന്ദർശിക്കുക.

## മുൻഗണനയുള്ള ഭാഷകൾ

എല്ലാ ആശയവിനിമയങ്ങളും ഇംഗ്ലീഷിൽ ആയിരിക്കണമെന്ന് ഞങ്ങൾ മുൻഗണന നൽകുന്നു.

## നയം

മൈക്രോസോഫ്റ്റ് [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd) എന്ന തത്വം പിന്തുടരുന്നു.
