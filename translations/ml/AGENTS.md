# AGENTS.md

## പ്രോജക്റ്റ് അവലോകനം

**Security-101** മൈക്രോസോഫ്റ്റ് സൃഷ്ടിച്ച ഒരു തുടക്കക്കാരൻ സൗഹൃദ സൈബർസുരക്ഷാ പാഠ്യപദ്ധതിയാണ്. ഈ പ്രോജക്റ്റ് ഘടനാപരമായ മോഡ്യൂളുകൾ വഴി അടിസ്ഥാന സൈബർസുരക്ഷാ ആശയങ്ങൾ പഠിപ്പിക്കുന്ന ഒരു ഡോക്യുമെന്റേഷൻ അടിസ്ഥാനമാക്കിയുള്ള പഠന വിഭവമാണ്. ഇത് വിൽപ്പനക്കാരനുമായി ബന്ധമില്ലാത്തതും ചെറിയ പാഠങ്ങൾ (ഓരോന്നും 30-60 മിനിറ്റ്) പൂർത്തിയാക്കാൻ രൂപകൽപ്പന ചെയ്തതുമാണ്.

**പ്രധാന സാങ്കേതികവിദ്യകൾ:**
- ഉള്ളടക്കത്തിനായി Markdown
- സ്റ്റാറ്റിക് സൈറ്റ് ജനറേഷനായി Docsify
- ഹോസ്റ്റിംഗിനായി GitHub Pages
- മൾട്ടി-ലാംഗ്വേജ് പിന്തുണയ്ക്കാൻ Co-op Translator (50+ ഭാഷകൾ)
- CI/CD-ക്കായി GitHub Actions

**ആർക്കിടെക്ചർ:**
- 8 പ്രധാന മോഡ്യൂളുകളായി ക്രമീകരിച്ച വിദ്യാഭ്യാസ ഉള്ളടക്കം, ഓരോന്നിനും ഉപപാഠങ്ങളുണ്ട്
- Markdown ഉള്ളടക്കം Docsify ഉപയോഗിച്ച് റെൻഡർ ചെയ്യുന്ന സ്റ്റാറ്റിക് HTML സൈറ്റ്
- Azure AI സേവനങ്ങൾ ഉപയോഗിച്ച് ഓട്ടോമേറ്റഡ് വിവർത്തന പ്രവൃത്തി
- കോർ ഉള്ളടക്കത്തിനായി ബിൽഡ് ടൂളുകളോ പാക്കേജ് മാനേജർമാരോ ആവശ്യമില്ല

## സെറ്റപ്പ് കമാൻഡുകൾ

ഇത് ഇൻസ്റ്റാൾ ചെയ്യാനുള്ള ആശ്രയങ്ങളില്ലാത്ത ഒരു ഡോക്യുമെന്റേഷൻ പ്രോജക്റ്റാണ്. ഉള്ളടക്കവുമായി പ്രവർത്തിക്കാൻ:

```bash
# റീപോസിറ്ററി ക്ലോൺ ചെയ്യുക
git clone https://github.com/microsoft/Security-101.git
cd Security-101

# ഉള്ളടക്കം ലോക്കലായി കാണുക - ഏതെങ്കിലും മാർക്ക്ഡൗൺ വ്യൂവർ പ്രവർത്തിക്കും
# അല്ലെങ്കിൽ ഡോക്സിഫൈ റെൻഡറിംഗ് ഉപയോഗിക്കാൻ ഒരു ലളിതമായ HTTP സർവർ ഉപയോഗിച്ച് സർവ് ചെയ്യുക
python -m http.server 8000
# തുടർന്ന് നിങ്ങളുടെ ബ്രൗസറിൽ http://localhost:8000 സന്ദർശിക്കുക
```

## ഡെവലപ്മെന്റ് പ്രവൃത്തി

### ഉള്ളടക്കം ലോക്കലായി കാണുക

പ്രോജക്റ്റ് Docsify ഉപയോഗിച്ച് റെൻഡറിംഗ് ചെയ്യുന്നു. മാറ്റങ്ങൾ പ്രിവ്യൂ ചെയ്യാൻ:

```bash
# ഓപ്ഷൻ 1: പൈതൺയുടെ ഇൻബിൽറ്റ് HTTP സെർവർ ഉപയോഗിക്കുക
python -m http.server 8000

# ഓപ്ഷൻ 2: നോഡ്.ജെഎസ് http-server ഉപയോഗിക്കുക (ലഭ്യമായാൽ)
npx http-server -p 8000

# ഓപ്ഷൻ 3: മാർക്ക്ഡൗൺ ഫയലുകൾ നേരിട്ട് ഏതെങ്കിലും മാർക്ക്ഡൗൺ എഡിറ്ററിൽ കാണുക
```

## കോഡ് സ്റ്റൈൽ മാർഗ്ഗനിർദ്ദേശങ്ങൾ

### Markdown കൺവെൻഷനുകൾ

- സ്റ്റാൻഡേർഡ് Markdown സിന്റാക്സ് ഉപയോഗിക്കുക
- തലക്കെട്ടുകൾ: പ്രധാന തലക്കെട്ടിന് `#`, വിഭാഗങ്ങൾക്ക് `##`, ഉപവിഭാഗങ്ങൾക്ക് `###` ഉപയോഗിക്കുക
- ലിസ്റ്റുകൾ: അൺഓർഡേർഡ് ലിസ്റ്റുകൾക്ക് `-` അല്ലെങ്കിൽ `*`, ഓർഡേർഡ് ലിസ്റ്റുകൾക്ക് `1.` ഉപയോഗിക്കുക
- ലിങ്കുകൾ: വിവരണാത്മകമായ ടെക്സ്റ്റ് ഉപയോഗിച്ച് പൂർണ്ണ GitHub URLs ഉപയോഗിക്കുക
- ചിത്രങ്ങൾ: `/images/` ഡയറക്ടറിയിൽ സൂക്ഷിക്കുക, വിവരണാത്മകമായ alt ടെക്സ്റ്റ് ഉപയോഗിക്കുക
- കോഡ് ബ്ലോക്കുകൾ: ആവശ്യമായപ്പോൾ ഭാഷാ ഐഡന്റിഫയർ ഉപയോഗിച്ച് ട്രിപ്പിൾ ബാക്ക്‌ടിക്കുകൾ ഉപയോഗിക്കുക

### ഉള്ളടക്ക മാർഗ്ഗനിർദ്ദേശങ്ങൾ

- പാഠങ്ങൾ ശ്രദ്ധാകേന്ദ്രവും സംക്ഷിപ്തവുമാക്കുക (30-60 മിനിറ്റ് വായന സമയം)
- വ്യക്തമായ, തുടക്കക്കാരൻ സൗഹൃദ ഭാഷ ഉപയോഗിക്കുക
- വിൽപ്പനക്കാരനുമായി ബന്ധപ്പെട്ട ഉപകരണ നിർദ്ദേശങ്ങൾ ഒഴിവാക്കുക (പാഠ്യപദ്ധതി വിൽപ്പനക്കാരനുമായി ബന്ധമില്ലാത്തതാണ്)

- അനുയോജ്യമായ സ്ഥലങ്ങളിൽ Microsoft Learn വിഭവങ്ങളിലേക്ക് ലിങ്ക് ചെയ്യുക
- ഉള്ളടക്കം വിദ്യാഭ്യാസപരമായതാണെന്ന് ഉറപ്പാക്കുക, പ്രമോഷണൽ അല്ല

**ഉദ്ദേശ്യം:** Jekyll ഉപയോഗിച്ച് സൈറ്റ് നിർമ്മിക്കുകയും ഡെപ്ലോയ് ചെയ്യുകയും ചെയ്യുന്നു

## പുൾ റിക്വസ്റ്റ് മാർഗ്ഗനിർദ്ദേശങ്ങൾ

### സമർപ്പിക്കുന്നതിന് മുമ്പ്

1. **ഉള്ളടക്ക അവലോകനം:** സൈബർസുരക്ഷാ ആശയങ്ങളുടെ കൃത്യതയും വ്യക്തതയും ഉറപ്പാക്കുക
2. **ഫോർമാറ്റിംഗ്:** Markdown ഫോർമാറ്റിംഗ് ശരിയായി റെൻഡർ ചെയ്യുന്നതായി സ്ഥിരീകരിക്കുക
3. **ലിങ്കുകൾ:** എല്ലാ ആന്തരിക, ബാഹ്യ ലിങ്കുകളും പരിശോധിക്കുക
4. **ചിത്രങ്ങൾ:** എല്ലാ ചിത്രങ്ങളും ലോഡ് ചെയ്യുകയും വിവരണാത്മകമായ alt ടെക്സ്റ്റ് ഉണ്ടാകുകയും ചെയ്യുന്നു
5. **സ്ഥിരത:** നിലവിലുള്ള ഉള്ളടക്ക ഘടനയും ശൈലിയും പാലിക്കുക

### PR തലക്കെട്ട് ഫോർമാറ്റ്

വിവരണാത്മകമായ തലക്കെട്ടുകൾ ഉപയോഗിക്കുക:
- `Add: [New content description]`
- `Update: [Module/Lesson] - [Brief description]`
- `Fix: [Issue description]`
- `Docs: [Documentation changes]`

### ആവശ്യമായ പരിശോധനകൾ

- ഉള്ളടക്ക കൃത്യത (മാനുവൽ അവലോകനം)
- Markdown ഫോർമാറ്റിംഗ് സാധൂകരണം
- ലിങ്ക് പരിശോധന
- വിവർത്തന പ്രവൃത്തി പൂർത്തിയാക്കൽ (`main` ബ്രാഞ്ച് മർജുകൾക്കായി)

### അവലോകന പ്രക്രിയ

1. കുറഞ്ഞത് ഒരു മെയിന്റെയ്‌നർ അവലോകനം ആവശ്യമാണ്
2. സൈബർസുരക്ഷാ ആശയങ്ങളുടെ സാങ്കേതിക കൃത്യതയിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കുക
3. ആക്സസിബിലിറ്റിയും തുടക്കക്കാരൻ സൗഹൃദത്വവും ഉറപ്പാക്കുക
4. വിൽപ്പനക്കാരനുമായി ബന്ധമില്ലാത്തതായ നില നിലനിർത്തുക

## സാധാരണ പ്രവർത്തനങ്ങൾ

### പുതിയ പാഠം ചേർക്കൽ

```bash
# 1. ശരിയായ പേരിടലോടെ പുതിയ മാർക്ക്ഡൗൺ ഫയൽ സൃഷ്ടിക്കുക
touch "[Module].[Lesson] [Title].md"

# 2. ടെംപ്ലേറ്റ് ഘടന അനുസരിച്ച് ഉള്ളടക്കം ചേർക്കുക
# 3. പുതിയ എൻട്രിയോടെ README.md മോഡ്യൂൾ പട്ടിക അപ്ഡേറ്റ് ചെയ്യുക
# 4. മോഡ്യൂൾ അവലോകന പട്ടികയിൽ എൻട്രി ചേർക്കുക
# 5. ക്വിസ് ഫയൽ നമ്പറിംഗ് ശരിയാണെന്ന് ഉറപ്പാക്കുക

# 6. മാറ്റങ്ങൾ കമ്മിറ്റ് ചെയ്യുക
git add "[Module].[Lesson] [Title].md" README.md
git commit -m "Add: [Module].[Lesson] - [Title]"
git push
```

### നിലവിലുള്ള ഉള്ളടക്കം അപ്ഡേറ്റ് ചെയ്യുക

```bash
# 1. അനുയോജ്യമായ മാർക്ക്ഡൗൺ ഫയൽ എഡിറ്റ് ചെയ്യുക
# 2. ഫോർമാറ്റിംഗ്, ലിങ്കുകൾ എന്നിവ പരിശോധിക്കുക
# 3. വിവരണാത്മകമായ സന്ദേശത്തോടെ കമ്മിറ്റ് ചെയ്യുക
git add "[Module].[Lesson] [Title].md"
git commit -m "Update: [Module].[Lesson] - [Brief description of changes]"
git push
```

### ചിത്രങ്ങൾ ചേർക്കൽ

```bash
# 1. ചിത്രം /images/ ഡയറക്ടറിയിലേക്ക് ചേർക്കുക
cp new-image.png images/

# 2. മാർക്ക്ഡൗണിൽ പരാമർശിക്കുക
# ![വിവരണാത്മകമായ ആൾട്ട് ടെക്സ്റ്റ്](../../translated_images/new-image.ade387a7c819665dda19f7585b9b1cc82817a21d283e286f39c2891d201fcdab.ml.png)

# 3. ഉള്ളടക്കവും ചിത്രവും ഒരുമിച്ച് കമ്മിറ്റ് ചെയ്യുക
git add "images/new-image.png" "[Module].[Lesson] [Title].md"
git commit -m "Add: Image for [description]"
git push
```

## പരിശോധനയും സാധൂകരണവും

### ഉള്ളടക്ക സാധൂകരണം

ഇത് ഒരു ഡോക്യുമെന്റേഷൻ പ്രോജക്റ്റായതിനാൽ, പരിശോധനയിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കുന്നത്:

1. **Markdown റെൻഡറിംഗ്:** മാറ്റങ്ങൾ ലോക്കലായി കാണുക (Docsify ഉപയോഗിച്ച്)
2. **ലിങ്ക് സാധൂകരണം:** എല്ലാ ലിങ്കുകളും മാനുവൽ ആയി ക്ലിക്ക് ചെയ്യുക
3. **ചിത്ര ലോഡിംഗ്:** എല്ലാ ചിത്രങ്ങളും ശരിയായി പ്രദർശിപ്പിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
4. **വിവർത്തനം:** ഫയലുകൾ വിവർത്തന പ്രവൃത്തിയാൽ പിടിച്ചെടുക്കുന്നുണ്ടെന്ന് പരിശോധിക്കുക (ഓട്ടോമേറ്റഡ്)
5. **ആക്സസിബിലിറ്റി:** ശരിയായ തലക്കെട്ട് ക്രമവും alt ടെക്സ്റ്റും ഉറപ്പാക്കുക

### ലോക്കൽ പ്രിവ്യൂ

```bash
# ഡോക്സിഫൈ റെൻഡറിംഗ് പരിശോധിക്കാൻ ലോക്കലായി സർവ് ചെയ്യുക
python -m http.server 8000

# http://localhost:8000 സന്ദർശിക്കുക
# എല്ലാ ലിങ്കുകളും പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കാൻ നാവിഗേഷൻ വഴി ക്ലിക്ക് ചെയ്യുക
# ചിത്രങ്ങൾ ശരിയായി ലോഡ് ചെയ്യുന്നുണ്ടെന്ന് പരിശോധിക്കുക
# മാർക്ക്ഡൗൺ ഫോർമാറ്റിംഗ് ശരിയായി റെൻഡർ ചെയ്യുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
```

## സാധാരണ പ്രശ്നങ്ങളും പരിഹാരങ്ങളും

### വിവർത്തനങ്ങൾ പ്രവർത്തിക്കുന്നില്ല

- മാറ്റങ്ങൾ `main` ബ്രാഞ്ചിലേക്ക് പുഷ് ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
- GitHub Actions ടാബിൽ പ്രവൃത്തി നില പരിശോധിക്കുക
- വിവർത്തന പ്രവൃത്തി രഹസ്യങ്ങൾ ക്രമീകരിച്ചിട്ടുണ്ടെന്ന് സ്ഥിരീകരിക്കുക
- വിവർത്തനം ഓട്ടോമാറ്റിക്കാണ്; വിവർത്തന ഫയലുകൾ കൈമാറ്റം ചെയ്യരുത്

### ചിത്രങ്ങൾ ലോഡാകുന്നില്ല

- ചിത്ര പാത്ത് ഫോർവേഡ് സ്ലാഷുകൾ ഉപയോഗിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക: `images/filename.png`
- ചിത്ര ഫയൽ റിപോസിറ്ററിയിൽ കമിറ്റ് ചെയ്തിട്ടുണ്ടെന്ന് പരിശോധിക്കുക
- ചിത്ര ഫയൽനാമം കൃത്യമായി പൊരുത്തപ്പെടുന്നു (കേസ്-സെൻസിറ്റീവ്)
- റിലേറ്റീവ് പാത്തുകൾ ഉപയോഗിക്കുക, ആബ്സല്യൂട്ട് URLs അല്ല

### Docsify റെൻഡർ ചെയ്യുന്നില്ല

- `index.html` റൂട്ട്-ൽ ഉണ്ടെന്ന് പരിശോധിക്കുക
- Docsify CDN ലിങ്കുകൾ ആക്സസിബിൾ ആണെന്ന് ഉറപ്പാക്കുക
- Markdown ഫയലുകൾ സ്റ്റാൻഡേർഡ് സിന്റാക്സ് ഉപയോഗിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
- ബ്രൗസർ കൺസോളിൽ ജാവാസ്ക്രിപ്റ്റ് പിശകുകൾ പരിശോധിക്കുക

### ലിങ്കുകൾ പ്രവർത്തിക്കുന്നില്ല

- പാഠങ്ങൾ തമ്മിലുള്ള ക്രോസ്-റഫറൻസുകൾക്കായി പൂർണ്ണ GitHub URLs ഉപയോഗിക്കുക
- ഫോർമാറ്റ്: `https://github.com/microsoft/Security-101/blob/main/[file].md`
- റോ മാർക്ക്ഡൗണിൽ മാത്രമല്ല, റെൻഡർ ചെയ്ത കാഴ്ചയിൽ ലിങ്കുകൾ പരിശോധിക്കുക

## പ്രോജക്റ്റ് പരിപാലനം

### സ്ഥിരമായ പ്രവർത്തനങ്ങൾ

- കമ്മ്യൂണിറ്റി സംഭാവനകൾ അവലോകനം ചെയ്യുകയും മർജ് ചെയ്യുകയും ചെയ്യുക
- സുരക്ഷാ രംഗം മാറുമ്പോൾ ഉള്ളടക്കം കൃത്യതയ്ക്കായി അപ്ഡേറ്റ് ചെയ്യുക
- വിവർത്തന പ്രവൃത്തി പിശകുകൾക്കായി നിരീക്ഷിക്കുക
- പ്രശ്നങ്ങൾക്കും ചർച്ചകൾക്കും പ്രതികരിക്കുക
- ബാഹ്യ ലിങ്കുകൾ അപ്ഡേറ്റ് ചെയ്യുക

### പതിപ്പ് നിയന്ത്രണം

- `main` ബ്രാഞ്ച് സംരക്ഷിതമാണ്, PR അവലോകനങ്ങൾ ആവശ്യമാണ്
- എല്ലാ മാറ്റങ്ങളും പുൾ റിക്വസ്റ്റ് പ്രക്രിയയിലൂടെ പോകുന്നു
- വിവർത്തന ഫയലുകൾ ഓട്ടോ-ജനറേറ്റഡ് ആണ്, കൈമാറ്റം ചെയ്യരുത്
- അർത്ഥവത്തായ കമിറ്റ് സന്ദേശങ്ങൾ ഉപയോഗിക്കുക

## AI കോഡിംഗ് ഏജന്റുകൾക്കുള്ള കുറിപ്പുകൾ

- **ഇത് ഒരു ഡോക്യുമെന്റേഷൻ പ്രോജക്റ്റാണ്** - ഉള്ളടക്ക ഗുണനിലവാരത്തിൽ ശ്രദ്ധ കേന്ദ്രീകരിക്കുക, കോഡിൽ അല്ല
- **വിവർത്തന ഫയലുകൾ മാറ്റരുത്** - അവ ഓട്ടോ-ജനറേറ്റഡ് ആണ്
- **ഫയൽ നാമകരണം കൺവെൻഷനുകൾ പാലിക്കുക** - നിലവിലുള്ള മാതൃക പ്രകാരം ഫയൽനാമങ്ങളിൽ സ്പേസുകൾ ഉപയോഗിക്കുക
- **README.md അപ്ഡേറ്റ് ചെയ്യുക** മോഡ്യൂളുകൾ ചേർക്കുകയോ നീക്കുകയോ ചെയ്യുമ്പോൾ പട്ടിക സിങ്ക് ചെയ്യാൻ
- **PR സമർപ്പിക്കുന്നതിന് മുമ്പ് ലോക്കലായി പരീക്ഷിക്കുക** Markdown ശരിയായി റെൻഡർ ചെയ്യുന്നതായി ഉറപ്പാക്കാൻ
- **നിലവിലുള്ള ഉള്ളടക്ക ശൈലി പാലിക്കുക** - തുടക്കക്കാരൻ സൗഹൃദവും വിൽപ്പനക്കാരനുമായി ബന്ധമില്ലാത്തതുമായ ശൈലി നിലനിർത്തുക
- **ബിൽഡ് പ്രക്രിയ ആവശ്യമില്ല** - ലോക്കൽ ഡെവലപ്മെന്റിനായി ലളിതമായ HTTP സെർവർ മതിയാകും
- **മോഡ്യൂൾ ഘടന മാനിക്കുക** - ഓരോ മോഡ്യൂളിനും സ്ഥിരമായ നമ്പറിംഗും ക്രമീകരണവും ഉണ്ട്
