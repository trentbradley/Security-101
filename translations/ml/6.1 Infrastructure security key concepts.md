# ഇൻഫ്രാസ്ട്രക്ചർ സുരക്ഷയുടെ പ്രധാന ആശയങ്ങൾ

[![വീഡിയോ കാണുക](../../translated_images/6-1_placeholder.773c176b8b7e3560d49a8ab481a9457006c04ad3c7b3acd4a4291af6da21df7f.ml.png)](https://learn-video.azurefd.net/vod/player?id=729d969e-c8ce-4889-aaa0-e5d92658ed62)

"ഇൻഫ്രാസ്ട്രക്ചർ" എന്നത് ഏതെങ്കിലും ഐടി പരിസ്ഥിതിയുടെ അടിസ്ഥാന ഘടകങ്ങളെ സൂചിപ്പിക്കുന്നു, ഇതിൽ സെർവറുകൾ, ക്ലൗഡ് സേവനങ്ങൾ, കണ്ടെയ്‌നറുകൾ എന്നിവ ഉൾപ്പെടുന്നു - വിവിധ സാങ്കേതികവിദ്യകളുടെ വലിയ ഒരു വിഭാഗം. മുൻവശം ചർച്ച ചെയ്ത ആപ്ലിക്കേഷനുകൾ ഇൻഫ്രാസ്ട്രക്ചറിൽ പ്രവർത്തിക്കുന്നു, അതിനാൽ ഇത് ആക്രമകരുടെ ലക്ഷ്യമായേക്കാം.

ഈ പാഠത്തിൽ, നാം ചർച്ച ചെയ്യുന്നത്:

- സുരക്ഷാ ശീലങ്ങൾ എന്താണ്, അത് എന്തുകൊണ്ട് പ്രധാനമാണ്?

- സുരക്ഷാ നിലനില്പ് മാനേജ്മെന്റ് എന്താണ്?

- പാച്ചിംഗ് എന്താണ്, അത് എന്തുകൊണ്ട് പ്രധാനമാണ്?

- കണ്ടെയ്‌നറുകൾ എന്താണ്, അവയുടെ സുരക്ഷയിൽ എന്താണ് വ്യത്യാസം?

## സുരക്ഷാ ശീലങ്ങൾ എന്താണ്, അത് എന്തുകൊണ്ട് പ്രധാനമാണ്?

**സുരക്ഷാ ശീലങ്ങൾ** എന്നത് വ്യക്തികളും സംഘടനകളും നല്ല സൈബർസുരക്ഷാ ശീലങ്ങൾ പാലിക്കാൻ പിന്തുടരുന്ന പ്രവർത്തനങ്ങളും പെരുമാറ്റങ്ങളും സൂചിപ്പിക്കുന്നു. ഇത് സിസ്റ്റങ്ങൾ, ഡാറ്റ, നെറ്റ്വർക്കുകൾ എന്നിവ ഭീഷണികളിൽ നിന്നും ദുർബലതകളിൽ നിന്നും സംരക്ഷിക്കാൻ നടപടികൾ സ്വീകരിക്കുന്നതിൽ ഉൾപ്പെടുന്നു. നല്ല സുരക്ഷാ ശീലങ്ങൾ പല കാരണങ്ങൾക്കായി പ്രധാനമാണ്:

- **ആക്രമണങ്ങൾ തടയൽ**: മികച്ച ശീലങ്ങൾ പിന്തുടരുന്നത് ഫിഷിംഗ്, മാൽവെയർ അണുബാധകൾ, അനധികൃത പ്രവേശനം തുടങ്ങിയ സാധാരണ സൈബർ ആക്രമണങ്ങൾ തടയാൻ സഹായിക്കുന്നു.

- **ഡാറ്റ സംരക്ഷണം**: ശരിയായ സുരക്ഷാ ശീലങ്ങൾ സംവേദനാത്മകവും രഹസ്യവുമായ ഡാറ്റ മോഷണത്തിൽ നിന്നും പുറത്തുപോകുന്നതിൽ നിന്നും സംരക്ഷിക്കുന്നു.

- **വിശ്വാസം നിലനിർത്തൽ**: നല്ല സുരക്ഷാ ശീലങ്ങൾ പ്രദർശിപ്പിക്കുന്ന സംഘടനകൾ ഉപഭോക്താക്കളുമായി പങ്കാളികളുമായി വിശ്വാസം നിർമ്മിക്കുന്നു.

- **അനുസരണം**: പല നിയമങ്ങളും മാനദണ്ഡങ്ങളും സംഘടനകൾ സുരക്ഷാ ശീലങ്ങൾ പാലിക്കണമെന്ന് ആവശ്യപ്പെടുന്നു.

- **റിസ്ക് കുറയ്ക്കൽ**: സ്ഥിരമായ സുരക്ഷാ ശീലങ്ങൾ സുരക്ഷാ സംഭവങ്ങളും ലംഘനങ്ങളും കുറയ്ക്കുന്നു.

സുരക്ഷാ ശീലങ്ങളിൽ സോഫ്റ്റ്വെയർ അപ്‌ഡേറ്റ് ചെയ്യൽ, ശക്തമായ പാസ്‌വേഡുകൾ ഉപയോഗിക്കൽ, മൾട്ടി-ഫാക്ടർ ഓതന്റിക്കേഷൻ, സ്ഥിരമായ ബാക്കപ്പ്, ജീവനക്കാരുടെ പരിശീലനം, സംശയാസ്പദമായ പ്രവർത്തനങ്ങൾ നിരീക്ഷിക്കൽ എന്നിവ ഉൾപ്പെടുന്നു. ഇത് ശക്തമായ സൈബർസുരക്ഷാ നിലനില്പിന്റെ അടിസ്ഥാനം ആണ്.

## സുരക്ഷാ നിലനില്പ് മാനേജ്മെന്റ് എന്താണ്?

സുരക്ഷാ നിലനില്പ് മാനേജ്മെന്റ് എന്നത് ഒരു സംഘടനയുടെ മൊത്തത്തിലുള്ള സൈബർസുരക്ഷാ നിലനില്പ് വിലയിരുത്തൽ, നിരീക്ഷിക്കൽ, മാനേജ്മെന്റ് എന്നിവയെ സൂചിപ്പിക്കുന്നു. സൈബർസുരക്ഷാ നിലനില്പ് എന്നത് ഒരു സംഘടനയുടെ ഐടി സിസ്റ്റങ്ങൾ, നെറ്റ്വർക്കുകൾ, ഡാറ്റ, ആസ്തികൾ എന്നിവ സൈബർ ഭീഷണികളിൽ നിന്നും ആക്രമണങ്ങളിൽ നിന്നും സംരക്ഷിക്കാൻ ഉള്ള മൊത്തത്തിലുള്ള സമീപനവും തയ്യാറെടുപ്പും ആണ്. ഇത് ഒരു സംഘടനയുടെ ഡിജിറ്റൽ ആസ്തികളെ സംരക്ഷിക്കാൻ ഉപയോഗിക്കുന്ന തന്ത്രങ്ങൾ, നയങ്ങൾ, ശീലങ്ങൾ, സാങ്കേതികവിദ്യകൾ എന്നിവ ഉൾക്കൊള്ളുന്നു. 

ഇത് സിസ്റ്റങ്ങൾ, നെറ്റ്വർക്കുകൾ, ആപ്ലിക്കേഷനുകൾ, ഡാറ്റ എന്നിവയുടെ സുരക്ഷാ നയങ്ങൾ, മികച്ച ശീലങ്ങൾ, അനുസരണ ആവശ്യങ്ങൾ എന്നിവയുമായി പൊരുത്തപ്പെടുന്നുവെന്ന് ഉറപ്പാക്കാൻ അവയുടെ സുരക്ഷാ നിലനില്പ് വിലയിരുത്തുകയും നിലനിർത്തുകയും ചെയ്യുന്നതിൽ ഉൾപ്പെടുന്നു. SPM ഒരു സംഘടനയുടെ സുരക്ഷാ നിലനില്പിന്റെ സമഗ്രമായ കാഴ്ചപ്പാട് നൽകുകയും, ദുർബലതകളും ദുർബലതകളും തിരിച്ചറിയുകയും, പരിഹാര ശ്രമങ്ങൾക്ക് മുൻഗണന നൽകുകയും ചെയ്യുന്നു.

## പാച്ചിംഗ് എന്താണ്, അത് എന്തുകൊണ്ട് പ്രധാനമാണ്?

**പാച്ചിംഗ്** എന്നത് സോഫ്റ്റ്വെയറുകൾ, ഓപ്പറേറ്റിംഗ് സിസ്റ്റങ്ങൾ, ആപ്ലിക്കേഷനുകൾ എന്നിവയ്ക്ക് പാച്ചുകൾ അല്ലെങ്കിൽ പരിഹാരങ്ങൾ എന്നറിയപ്പെടുന്ന സോഫ്റ്റ്വെയർ അപ്‌ഡേറ്റുകൾ പ്രയോഗിക്കുന്ന പ്രക്രിയയെ സൂചിപ്പിക്കുന്നു. ഈ അപ്‌ഡേറ്റുകൾ സാധാരണയായി സുരക്ഷാ ദുർബലതകൾ, ബഗുകൾ, ആക്രമകരാൽ ഉപയോഗപ്പെടുത്താവുന്ന മറ്റ് പ്രശ്നങ്ങൾ എന്നിവ പരിഹരിക്കുന്നു. ഹാർഡ്‌വെയർ ഉപകരണങ്ങൾക്കും പാച്ചിംഗ് ആവശ്യമാണ്: ഇത് അവരുടെ ഫർമ്വെയർ അല്ലെങ്കിൽ ബിൽറ്റ്-ഇൻ ഓഎസ് ആയിരിക്കാം. ഹാർഡ്‌വെയർ പാച്ചിംഗ് സോഫ്റ്റ്വെയർ പാച്ചിംഗിനേക്കാൾ കൂടുതൽ പ്രയാസകരമായിരിക്കും.

പാച്ചിംഗ് പ്രധാനമാണ് പല കാരണങ്ങൾക്കായി:

- **സുരക്ഷ**: പാച്ചുകൾ ആക്രമകർ സിസ്റ്റങ്ങൾ കൈവശപ്പെടുത്താനും ഡാറ്റ മോഷ്ടിക്കാനും ഉപയോഗിക്കുന്ന ദുർബലതകൾ പരിഹരിക്കുന്നു.

- **സ്ഥിരത**: പാച്ചുകൾ സ്ഥിരതയും പ്രകടന മെച്ചപ്പെടുത്തലുകളും ഉൾക്കൊള്ളുന്നു, ക്രാഷുകൾ അല്ലെങ്കിൽ സിസ്റ്റം തകരാറുകൾ കുറയ്ക്കുന്നു.

- **അനുസരണം**: പല നിയമങ്ങളും അനുസരണ മാനദണ്ഡങ്ങളും സംഘടനകൾ സുരക്ഷാ പാച്ചുകൾ ഉടൻ പ്രയോഗിക്കണമെന്ന് ആവശ്യപ്പെടുന്നു.

- **വിശ്വാസം നിലനിർത്തൽ**: സ്ഥിരമായ പാച്ചിംഗ് ഉപഭോക്താക്കളുടെയും പങ്കാളികളുടെയും വിശ്വാസം നിലനിർത്താൻ സഹായിക്കുന്നു, സുരക്ഷയോടുള്ള പ്രതിബദ്ധത പ്രദർശിപ്പിക്കുന്നു.

- **റിസ്ക് കുറയ്ക്കൽ**: പാച്ചിംഗ് ആക്രമണ സാധ്യതയും വിജയകരമായ സൈബർ ആക്രമണങ്ങളുടെ സാധ്യതയും കുറയ്ക്കുന്നു.

പാച്ചുകൾ സമയബന്ധിതമായി പ്രയോഗിക്കാത്തത് സിസ്റ്റങ്ങൾ അറിയപ്പെടുന്ന ഉപയോഗങ്ങളിൽ ദുർബലമാക്കുകയും, സുരക്ഷാ ലംഘനങ്ങളും ഡാറ്റ നഷ്ടവും വർദ്ധിപ്പിക്കുകയും ചെയ്യുന്നു.

## കണ്ടെയ്‌നറുകൾ എന്താണ്, അവയുടെ സുരക്ഷയിൽ എന്താണ് വ്യത്യാസം?

കണ്ടെയ്‌നറുകൾ എന്നത് ഒരു സോഫ്റ്റ്വെയർ പ്രവർത്തിപ്പിക്കാൻ ആവശ്യമായ കോഡ്, റൺടൈം, ലൈബ്രറികൾ, സിസ്റ്റം ടൂളുകൾ എന്നിവ ഉൾക്കൊള്ളുന്ന ലഘുവായ, സ്വതന്ത്രമായ, പ്രവർത്തനക്ഷമമായ സോഫ്റ്റ്വെയർ പാക്കേജുകളാണ്. കണ്ടെയ്‌നറുകൾ ആപ്ലിക്കേഷനുകൾക്കായി സ്ഥിരതയുള്ള, വേർതിരിച്ച പരിസ്ഥിതികൾ നൽകുന്നു, വിവിധ പരിസ്ഥിതികളിലും പ്ലാറ്റ്ഫോമുകളിലും സോഫ്റ്റ്വെയർ വികസിപ്പിക്കാനും പാക്കേജുചെയ്യാനും വിന്യസിക്കാനും എളുപ്പമാക്കുന്നു. പ്രശസ്തമായ കണ്ടെയ്‌നറൈസേഷൻ സാങ്കേതികവിദ്യകളിൽ Docker, Kubernetes എന്നിവ ഉൾപ്പെടുന്നു.

കണ്ടെയ്‌നർ സുരക്ഷ എന്നത് കണ്ടെയ്‌നറുകളെയും അവയിൽ ഹോസ്റ്റ് ചെയ്യുന്ന ആപ്ലിക്കേഷനുകളെയും വിവിധ സുരക്ഷാ ഭീഷണികളിൽ നിന്നും ദുർബലതകളിൽ നിന്നും സംരക്ഷിക്കാൻ ഉപയോഗിക്കുന്ന ശീലങ്ങളും സാങ്കേതികവിദ്യകളും ആണ്. കണ്ടെയ്‌നർ സുരക്ഷ പ്രധാനമാണ്, കാരണം കണ്ടെയ്‌നറുകൾ പോർട്ടബിലിറ്റിയിലും സ്കെയിലബിലിറ്റിയിലും പല ഗുണങ്ങളും നൽകുമ്പോഴും, അവ സുരക്ഷാ വെല്ലുവിളികൾ അവതരിപ്പിക്കുന്നു:

1. **ഇമേജ് സുരക്ഷ:** കണ്ടെയ്‌നർ ഇമേജുകളിൽ ദുർബലതകൾ ഉണ്ടാകാം, ഈ ഇമേജുകൾ സ്ഥിരമായി അപ്‌ഡേറ്റ് ചെയ്യുകയും പാച്ച് ചെയ്യുകയും ചെയ്യാത്ത പക്ഷം, അവ ആക്രമകരാൽ ഉപയോഗപ്പെടുത്തപ്പെടാം. കണ്ടെയ്‌നർ സുരക്ഷയിൽ അറിയപ്പെടുന്ന ദുർബലതകൾക്കായി ഇമേജുകൾ സ്കാൻ ചെയ്യുകയും വിശ്വസനീയമായ ഇമേജുകൾ മാത്രം ഉപയോഗിക്കുകയുമാണ്.

2. **റൺടൈം സുരക്ഷ:** പ്രവർത്തിക്കുന്ന കണ്ടെയ്‌നറുകൾ പരസ്പരം വേർതിരിക്കപ്പെടുകയും ഹോസ്റ്റ് സിസ്റ്റത്തിൽ നിന്നും വേർതിരിക്കപ്പെടുകയും വേണം, അനധികൃത പ്രവേശനവും സാധ്യതയുള്ള ആക്രമണങ്ങളും തടയാൻ. റൺടൈം സുരക്ഷാ സംവിധാനങ്ങളിൽ namespaces, cgroups പോലുള്ള കണ്ടെയ്‌നർ വേർതിരിക്കൽ സാങ്കേതികവിദ്യകളും, കണ്ടെയ്‌നർ പെരുമാറ്റം നിരീക്ഷിക്കുന്നതിനും ഓഡിറ്റുചെയ്യുന്നതിനും ഉപകരണങ്ങളും ഉൾപ്പെടുന്നു.

3. **നെറ്റ്വർക്ക് സുരക്ഷ:** കണ്ടെയ്‌നറുകൾ പരസ്പരം, ബാഹ്യ സിസ്റ്റങ്ങളുമായി നെറ്റ്വർക്കുകൾ വഴി ആശയവിനിമയം നടത്തുന്നു. കണ്ടെയ്‌നറുകൾ തമ്മിലുള്ള ട്രാഫിക് നിയന്ത്രിക്കാൻ, അനധികൃത പ്രവേശനം തടയാൻ ശരിയായ നെറ്റ്വർക്ക് സെഗ്മെന്റേഷൻ, ഫയർവാൾ നിയമങ്ങൾ അനിവാര്യമാണ്.

4. **ആക്സസ് നിയന്ത്രണം:** അനധികൃത ഉപയോക്താക്കളും പ്രക്രിയകളും കണ്ടെയ്‌നറുകൾക്ക് പ്രവേശിക്കുകയും മാറ്റം വരുത്തുകയും ചെയ്യുന്നത് തടയുന്നത് നിർണായകമാണ്. Role-based access control (RBAC) പോലുള്ള ഉപകരണങ്ങളും ഐഡന്റിറ്റി മാനേജ്മെന്റ് ഉപകരണങ്ങളും സാധാരണയായി ആക്സസ് നിയന്ത്രണത്തിന് ഉപയോഗിക്കുന്നു.

5. **ലോഗിംഗ്, നിരീക്ഷണം:** കണ്ടെയ്‌നർ സുരക്ഷയിൽ ലോഗുകൾ ശേഖരിക്കുകയും, സുരക്ഷാ സംഭവങ്ങളും അനിഷ്ടതകളും റിയൽ-ടൈമിൽ കണ്ടെത്താനും പ്രതികരിക്കാനും നിരീക്ഷണ ഡാറ്റ വിശകലനം ചെയ്യുകയും ചെയ്യുന്നു.

6. **ഓർക്കസ്ട്രേഷൻ സുരക്ഷ:** Kubernetes പോലുള്ള കണ്ടെയ്‌നർ ഓർക്കസ്ട്രേഷൻ പ്ലാറ്റ്ഫോമുകൾ ഉപയോഗിക്കുമ്പോൾ, ഓർക്കസ്ട്രേഷൻ ലെയർ സുരക്ഷിതമാക്കുന്നത് തുല്യമായി പ്രധാനമാണ്. ഇതിൽ Kubernetes API സെർവർ സുരക്ഷിതമാക്കൽ, ശരിയായ RBAC നയങ്ങൾ ഉറപ്പാക്കൽ, ക്ലസ്റ്റർ പ്രവർത്തനങ്ങൾ ഓഡിറ്റ് ചെയ്യൽ എന്നിവ ഉൾപ്പെടുന്നു.

7. **സീക്രെറ്റ്സ് മാനേജ്മെന്റ്:** API കീകൾ, പാസ്‌വേഡുകൾ പോലുള്ള സംവേദനാത്മക വിവരങ്ങൾ കണ്ടെയ്‌നറുകളിൽ കൈകാര്യം ചെയ്യുന്നത്, അവ പുറത്തുപോകുന്നത് തടയാൻ സുരക്ഷിത സംഭരണവും മാനേജ്മെന്റും ആവശ്യമാണ്.

കണ്ടെയ്‌നർ സുരക്ഷാ പരിഹാരങ്ങൾ സാധാരണയായി മികച്ച സുരക്ഷാ ശീലങ്ങൾ, ദുർബലതകൾ സ്കാൻ ചെയ്യാനുള്ള ഉപകരണങ്ങൾ, റൺടൈം സംരക്ഷണ സംവിധാനങ്ങൾ, നെറ്റ്വർക്ക് സുരക്ഷാ ക്രമീകരണങ്ങൾ, കണ്ടെയ്‌നർ ഓർക്കസ്ട്രേഷൻ സുരക്ഷാ സവിശേഷതകൾ എന്നിവയുടെ സംയോജനം ഉൾക്കൊള്ളുന്നു. കണ്ടെയ്‌നറൈസ്ഡ് ആപ്ലിക്കേഷനുകൾ വികസിക്കുകയും സ്കെയിൽ ചെയ്യുകയും ചെയ്യുന്നതിനാൽ ഭീഷണികളെ വേഗത്തിൽ കണ്ടെത്താനും പ്രതികരിക്കാനും തുടർച്ചയായ നിരീക്ഷണവും ഓട്ടോമേഷൻ സംവിധാനങ്ങളും നിർണായക ഘടകങ്ങളാണ്.
