# AGENTS.md

## ప్రాజెక్ట్ అవలోకనం

**సెక్యూరిటీ-101** అనేది మైక్రోసాఫ్ట్ రూపొందించిన ఒక ప్రారంభ స్థాయి సైబర్‌సెక్యూరిటీ పాఠ్యాంశం. ఈ ప్రాజెక్ట్ ఒక డాక్యుమెంటేషన్ ఆధారిత లెర్నింగ్ రిసోర్స్, ఇది నిర్మితమైన మాడ్యూల్స్ ద్వారా ప్రాథమిక సైబర్‌సెక్యూరిటీ కాన్సెప్ట్‌లను నేర్పుతుంది. ఇది విక్రేత-నిరపేక్షంగా ఉంటుంది మరియు చిన్న పాఠాలుగా (ప్రతి ఒక్కటి 30-60 నిమిషాలు) పూర్తి చేయడానికి రూపొందించబడింది.

**ప్రధాన సాంకేతికతలు:**
- కంటెంట్ కోసం మార్క్‌డౌన్
- స్థిరమైన సైట్ జనరేషన్ కోసం Docsify
- హోస్టింగ్ కోసం GitHub Pages
- బహుభాషా మద్దతు కోసం Co-op Translator (50+ భాషలు)
- CI/CD కోసం GitHub Actions

**ఆర్కిటెక్చర్:**
- 8 ప్రధాన మాడ్యూల్స్‌లో విద్యా కంటెంట్‌ను క్రమబద్ధంగా ఏర్పాటు చేయడం, ప్రతి మాడ్యూల్‌లో ఉపపాఠాలు ఉన్నాయి
- Docsify ద్వారా మార్క్‌డౌన్ కంటెంట్‌ను రెండర్ చేసే స్థిరమైన HTML సైట్
- Azure AI సేవలను ఉపయోగించి ఆటోమేటెడ్ అనువాద వర్క్‌ఫ్లో
- కోర్ కంటెంట్ కోసం బిల్డ్ టూల్స్ లేదా ప్యాకేజ్ మేనేజర్‌లు అవసరం లేదు

## సెటప్ కమాండ్లు

ఇది డాక్యుమెంటేషన్ ప్రాజెక్ట్, అందువల్ల ఇన్‌స్టాల్ చేయడానికి ఎలాంటి డిపెండెన్సీలు అవసరం లేదు. కంటెంట్‌తో పని చేయడానికి:

```bash
# రిపోజిటరీని క్లోన్ చేయండి
git clone https://github.com/microsoft/Security-101.git
cd Security-101

# లోకల్‌గా కంటెంట్‌ను వీక్షించండి - ఏదైనా మార్క్‌డౌన్ వీయర్ పనిచేస్తుంది
# లేదా డాక్సిఫై రెండరింగ్‌ను ఉపయోగించడానికి సింపుల్ HTTP సర్వర్‌తో సర్వ్ చేయండి
python -m http.server 8000
# ఆపై మీ బ్రౌజర్‌లో http://localhost:8000 ను సందర్శించండి
```

# ఎంపిక 3: ఏదైనా మార్క్‌డౌన్ ఎడిటర్‌లో మార్క్‌డౌన్ ఫైళ్లను నేరుగా వీక్షించండి

## కోడ్ స్టైల్ మార్గదర్శకాలు

### మార్క్‌డౌన్ కన్వెన్షన్‌లు

- ప్రామాణిక మార్క్‌డౌన్ సింటాక్స్‌ను ఉపయోగించండి
- శీర్షికలు: ప్రధాన శీర్షిక కోసం `#`, విభాగాల కోసం `##`, ఉపవిభాగాల కోసం `###` ఉపయోగించండి
- జాబితాలు: అన్‌ఆర్డర్డ్ జాబితాల కోసం `-` లేదా `*`, ఆర్డర్డ్ జాబితాల కోసం `1.` ఉపయోగించండి
- లింక్‌లు: వివరణాత్మక టెక్స్ట్‌తో పూర్తి GitHub URLలను ఉపయోగించండి
- చిత్రాలు: `/images/` డైరెక్టరీలో నిల్వ చేయండి, వివరణాత్మక alt టెక్స్ట్‌ను ఉపయోగించండి
- కోడ్ బ్లాక్‌లు: అవసరమైతే భాష గుర్తింపుతో ట్రిపుల్ బ్యాక్టిక్స్‌ను ఉపయోగించండి

### కంటెంట్ మార్గదర్శకాలు

- పాఠాలను కేంద్రీకృతంగా మరియు సంక్షిప్తంగా ఉంచండి (30-60 నిమిషాల చదవగలిగే సమయం)
- స్పష్టమైన, ప్రారంభ స్థాయి భాషను ఉపయోగించండి
- విక్రేత-నిరపేక్ష టూల్ సూచనలను నివారించండి (పాఠ్యాంశం విక్రేత-నిరపేక్షంగా ఉంటుంది)
- ప్రతి మాడ్యూల్ ప్రారంభంలో లెర్నింగ్ ఆబ్జెక్టివ్‌లను చేర్చండి
- అవసరమైనప్పుడు Microsoft Learn రిసోర్స్‌లకు లింక్ చేయండి
- కంటెంట్ విద్యా పరమైనదిగా ఉండాలి, ప్రమోషనల్‌గా కాదు

4. **చిత్రాలు:** అన్ని చిత్రాలు లోడ్ అవుతున్నాయా మరియు వివరణాత్మక alt టెక్స్ట్ ఉందా అని నిర్ధారించండి
5. **సమగ్రత:** ఉన్న కంటెంట్ నిర్మాణం మరియు శైలిని అనుసరించండి

### PR శీర్షిక ఫార్మాట్

వివరణాత్మక శీర్షికలను ఉపయోగించండి:
- `Add: [కొత్త కంటెంట్ వివరణ]`
- `Update: [Module/Lesson] - [సంక్షిప్త వివరణ]`
- `Fix: [ఇష్యూ వివరణ]`
- `Docs: [డాక్యుమెంటేషన్ మార్పులు]`

### అవసరమైన చెక్స్

- కంటెంట్ ఖచ్చితత్వం (మానవీయ రివ్యూ)
- మార్క్‌డౌన్ ఫార్మాటింగ్ ధృవీకరణ
- లింక్ ధృవీకరణ
- అనువాద వర్క్‌ఫ్లో పూర్తి (for `main` బ్రాంచ్ మర్జ్‌లు)

### రివ్యూ ప్రాసెస్

1. కనీసం ఒక మెయింటైనర్ రివ్యూ అవసరం
2. సెక్యూరిటీ కాన్సెప్ట్‌ల సాంకేతిక ఖచ్చితత్వంపై దృష్టి పెట్టండి
3. యాక్సెసిబిలిటీ మరియు ప్రారంభ స్థాయి అనుకూలతను ధృవీకరించండి
4. విక్రేత-నిరపేక్షతను కొనసాగించండి

## పరీక్ష మరియు ధృవీకరణ

### కంటెంట్ ధృవీకరణ

ఇది డాక్యుమెంటేషన్ ప్రాజెక్ట్ కాబట్టి, పరీక్ష ప్రధానంగా కింది అంశాలపై ఉంటుంది:

1. **మార్క్‌డౌన్ రెండరింగ్:** Docsify ఉపయోగించి లోకల్‌గా మార్పులను వీక్షించండి
2. **లింక్ ధృవీకరణ:** అన్ని లింక్‌లను మానవీయంగా క్లిక్ చేయండి
3. **చిత్రాలు లోడ్ అవుతాయా:** అన్ని చిత్రాలు సరిగా ప్రదర్శించబడుతున్నాయా అని ధృవీకరించండి
4. **అనువాదం:** ఫైల్‌లు అనువాద వర్క్‌ఫ్లో ద్వారా పికప్ చేయబడుతున్నాయా అని తనిఖీ చేయండి (ఆటోమేటెడ్)
5. **యాక్సెసిబిలిటీ:** సరైన శీర్షిక శ్రేణి మరియు alt టెక్స్ట్‌ను నిర్ధారించండి

### లోకల్ ప్రివ్యూ

```bash
# Docsify రెండరింగ్ పరీక్షించడానికి స్థానికంగా సర్వ్ చేయండి
python -m http.server 8000

# http://localhost:8000 సందర్శించండి
# అన్ని లింకులు పనిచేస్తున్నాయా అని ధృవీకరించడానికి నావిగేషన్ ద్వారా క్లిక్ చేయండి
# చిత్రాలు సరిగ్గా లోడ్ అవుతున్నాయా అని తనిఖీ చేయండి
# Markdown ఫార్మాటింగ్ సరిగ్గా రెండర్ అవుతుందా అని ధృవీకరించండి
```

### అనువాదాలు పనిచేయడం లేదు

- మార్పులు `main` బ్రాంచ్‌కు పుష్ చేయబడ్డాయా అని నిర్ధారించండి
- GitHub Actions ట్యాబ్‌లో వర్క్‌ఫ్లో స్థితిని తనిఖీ చేయండి
- అనువాద వర్క్‌ఫ్లో సీక్రెట్‌లు కాన్ఫిగర్ చేయబడ్డాయా అని ధృవీకరించండి

### చిత్రాలు లోడ్ కావడం లేదు

- చిత్ర పాత్ ఫార్వర్డ్ స్లాష్‌లను ఉపయోగిస్తుందా అని ధృవీకరించండి: `images/filename.png`
- చిత్ర ఫైల్ రిపోజిటరీకి కమిట్ చేయబడిందా అని తనిఖీ చేయండి
- చిత్ర ఫైల్ నేమ్ ఖచ్చితంగా సరిపోతుందా (కేస్-సెన్సిటివ్) అని నిర్ధారించండి
- రిలేటివ్ పాత్‌లను ఉపయోగించండి, అబ్సల్యూట్ URLలను కాదు

### Docsify రెండర్ చేయడం లేదు

- రూట్‌లో `index.html` ఉందా అని తనిఖీ చేయండి
- Docsify CDN లింక్‌లు యాక్సెసిబుల్‌గా ఉన్నాయా అని ధృవీకరించండి
- మార్క్‌డౌన్ ఫైల్‌లు ప్రామాణిక సింటాక్స్‌ను ఉపయోగిస్తున్నాయా అని నిర్ధారించండి
- బ్రౌజర్ కన్సోల్‌లో జావాస్క్రిప్ట్ లోపాలను తనిఖీ చేయండి

- అన్ని మార్పులు పుల్ రిక్వెస్ట్ ప్రాసెస్ ద్వారా వెళ్తాయి
- అనువాద ఫైల్‌లు ఆటో-జనరేట్ చేయబడతాయి, మానవీయంగా ఎడిట్ చేయవద్దు
- అర్థవంతమైన కమిట్ సందేశాలను ఉపయోగించండి

## AI కోడింగ్ ఏజెంట్లకు గమనికలు

- **ఇది డాక్యుమెంటేషన్ ప్రాజెక్ట్** - కంటెంట్ నాణ్యతపై దృష్టి పెట్టండి, కోడ్‌పై కాదు
- **అనువాద ఫైల్‌లను మార్చవద్దు** - అవి ఆటో-జనరేట్ చేయబడతాయి
- **ఫైల్ నేమింగ్ కన్వెన్షన్‌లను కాపాడండి** - ఉన్న నమూనా ప్రకారం ఫైల్ నేమ్‌లలో స్పేస్‌లను ఉపయోగించండి
- **README.mdని అప్‌డేట్ చేయండి** మాడ్యూల్స్ జోడించడం/తొలగించడం చేస్తే టేబుల్ సింక్‌లో ఉంచండి
- **PRలను సమర్పించే ముందు లోకల్‌గా పరీక్షించండి** మార్క్‌డౌన్ సరిగా రెండర్ అవుతుందా అని నిర్ధారించండి
- **ఉన్న కంటెంట్ శైలిని అనుసరించండి** - ప్రారంభ స్థాయి అనుకూలమైన, విక్రేత-నిరపేక్ష టోన్‌ను కొనసాగించండి
- **బిల్డ్ ప్రాసెస్ అవసరం లేదు** - లోకల్ డెవలప్‌మెంట్ కోసం సింపుల్ HTTP సర్వర్ సరిపోతుంది
- **మాడ్యూల్ నిర్మాణాన్ని గౌరవించండి** - ప్రతి మాడ్యూల్‌లో స్థిరమైన నంబరింగ్ మరియు ఆర్గనైజేషన్ ఉంటుంది
