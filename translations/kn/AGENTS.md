# AGENTS.md

## ಯೋಜನೆಯ ಅವಲೋಕನ

**Security-101** ಎಂಬುದು ಮೈಕ್ರೋಸಾಫ್ಟ್ ರಚಿಸಿದ ಪ್ರಾರಂಭಿಕ ಮಟ್ಟದ ಸೈಬರ್‌ಸುರಕ್ಷತಾ ಪಾಠಕ್ರಮವಾಗಿದೆ. ಈ ಯೋಜನೆವು ರಚನೆಗೊಳಿಸಿದ ಪಾಠಗಳ ಮೂಲಕ ಮೂಲಭೂತ ಸೈಬರ್‌ಸುರಕ್ಷತಾ ತತ್ವಗಳನ್ನು ಕಲಿಸುವ ಡಾಕ್ಯುಮೆಂಟ್ ಆಧಾರಿತ ಕಲಿಕೆಯ ಸಂಪತ್ತಾಗಿದೆ. ಇದು ಯಾವುದೇ ವಿಕ್ರೇತರಿಗೆ ಸಂಬಂಧಿಸದಂತೆ ವಿನ್ಯಾಸಗೊಳಿಸಲಾಗಿದೆ ಮತ್ತು 30-60 ನಿಮಿಷಗಳ ಪಾಠಗಳಲ್ಲಿ ಪೂರ್ಣಗೊಳ್ಳುವಂತೆ ರೂಪಿಸಲಾಗಿದೆ.

**ಮುಖ್ಯ ತಂತ್ರಜ್ಞಾನಗಳು:**
- ವಿಷಯಕ್ಕಾಗಿ Markdown
- Docsify ಮೂಲಕ ಸ್ಥಿರ ಸೈಟ್ ತಯಾರಿಕೆ
- GitHub Pages ಮೂಲಕ ಹೋಸ್ಟಿಂಗ್
- Co-op Translator ಮೂಲಕ ಬಹುಭಾಷಾ ಬೆಂಬಲ (50+ ಭಾಷೆಗಳು)
- GitHub Actions ಮೂಲಕ CI/CD

**ಆರ್ಕಿಟೆಕ್ಚರ್:**
- 8 ಮುಖ್ಯ ಮಾದರಿಗಳಲ್ಲಿ ಶೈಕ್ಷಣಿಕ ವಿಷಯವನ್ನು ಸಂಘಟಿಸಲಾಗಿದೆ, ಪ್ರತಿ ಮಾದರಿಯಲ್ಲೂ ಉಪಪಾಠಗಳು
- Docsify ಮೂಲಕ Markdown ವಿಷಯವನ್ನು ರೆಂಡರ್ ಮಾಡುವ ಸ್ಥಿರ HTML ಸೈಟ್
- Azure AI ಸೇವೆಗಳನ್ನು ಬಳಸುವ ಸ್ವಯಂಚಾಲಿತ ಅನುವಾದ ಕಾರ್ಯಪ್ರವೃತ್ತಿ
- ಮೂಲ ವಿಷಯಕ್ಕಾಗಿ ಯಾವುದೇ ಬಿಲ್ಡ್ ಟೂಲ್ಸ್ ಅಥವಾ ಪ್ಯಾಕೇಜ್ ಮ್ಯಾನೇಜರ್‌ಗಳ ಅಗತ್ಯವಿಲ್ಲ

## ಸೆಟಪ್ ಆಜ್ಞೆಗಳು

ಈ ಡಾಕ್ಯುಮೆಂಟ್ ಯೋಜನೆಗೆ ಯಾವುದೇ ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಲು ಅಗತ್ಯವಿಲ್ಲ. ವಿಷಯದೊಂದಿಗೆ ಕೆಲಸ ಮಾಡಲು:

```bash

git clone https://github.com/microsoft/Security-101.git

# ಅಥವಾ ಡಾಕ್ಸಿಫೈ ರೆಂಡರಿಂಗ್ ಬಳಸಲು ಸರಳ HTTP ಸರ್ವರ್‌ನೊಂದಿಗೆ ಸೇವೆ ಮಾಡಿ

# ನಂತರ ನಿಮ್ಮ ಬ್ರೌಸರ್‌ನಲ್ಲಿ http://localhost:8000 ಗೆ ಭೇಟಿ ನೀಡಿ
```

## ಕೋಡ್ ಶೈಲಿ ಮಾರ್ಗಸೂಚಿಗಳು

### Markdown ಸಂಪ್ರದಾಯಗಳು

- ಮಾನಕ Markdown ಶೈಲಿಯನ್ನು ಬಳಸಿ
- ಶೀರ್ಷಿಕೆಗಳು: ಮುಖ್ಯ ಶೀರ್ಷಿಕೆಗಾಗಿ `#`, ವಿಭಾಗಗಳಿಗಾಗಿ `##`, ಉಪವಿಭಾಗಗಳಿಗಾಗಿ `###` ಬಳಸಿ
- ಪಟ್ಟಿ: ಅಸಂಬದ್ಧ ಪಟ್ಟಿಗಾಗಿ `-` ಅಥವಾ `*`, ಕ್ರಮಬದ್ಧ ಪಟ್ಟಿಗಾಗಿ `1.` ಬಳಸಿ
- ಲಿಂಕ್: ವಿವರಣಾತ್ಮಕ ಪಠ್ಯವನ್ನು ಸಂಪೂರ್ಣ GitHub URL‌ಗಳೊಂದಿಗೆ ಬಳಸಿ
- ಚಿತ್ರಗಳು: `/images/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ, ವಿವರಣಾತ್ಮಕ alt ಪಠ್ಯವನ್ನು ಬಳಸಿ
- ಕೋಡ್ ಬ್ಲಾಕ್: ಅಗತ್ಯವಿದ್ದಲ್ಲಿ ಭಾಷಾ ಗುರುತಿನೊಂದಿಗೆ ತ್ರಿಪಲ್ ಬ್ಯಾಕ್ಟಿಕ್‌ಗಳನ್ನು ಬಳಸಿ

### ವಿಷಯ ಮಾರ್ಗಸೂಚಿಗಳು

- ಪಾಠಗಳನ್ನು ಕೇಂದ್ರೀಕೃತ ಮತ್ತು ಸಂಕ್ಷಿಪ್ತವಾಗಿಡಿ (30-60 ನಿಮಿಷದ ಓದು ಸಮಯ)
- ಸ್ಪಷ್ಟ, ಪ್ರಾರಂಭಿಕ ಮಟ್ಟದ ಭಾಷೆಯನ್ನು ಬಳಸಿ
- ವಿಕ್ರೇತರಿಗೆ ಸಂಬಂಧಿಸಿದ ಸಾಧನ ಸೂಚನೆಗಳನ್ನು ತಪ್ಪಿಸಿ (ಪಾಠಕ್ರಮವು ವಿಕ್ರೇತ-ಅಗ್ನೋಸ್ಟಿಕ್ ಆಗಿದೆ)
- ಪ್ರತಿ ಮಾಡ್ಯೂಲ್ ಪ್ರಾರಂಭದಲ್ಲಿ ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳನ್ನು ಸೇರಿಸಿ
- ಸೂಕ್ತವಾದ ಸ್ಥಳದಲ್ಲಿ Microsoft Learn ಸಂಪತ್ತಿಗೆ ಲಿಂಕ್ ಮಾಡಿ
- ವಿಷಯವು ಶೈಕ್ಷಣಿಕವಾಗಿರಬೇಕು, ಪ್ರಚಾರಾತ್ಮಕವಾಗಿರಬಾರದು

### ಫೈಲ್ ಹೆಸರಿಸುವುದು

- ಸ್ವರೂಪವನ್ನು ಬಳಸಿ: `[Module].[Lesson] [Title].md`

1. **ವಿಷಯ ಪರಿಶೀಲನೆ:** ಸೈಬರ್‌ಸುರಕ್ಷತಾ ತತ್ವಗಳ ಶುದ್ಧತೆ ಮತ್ತು ಸ್ಪಷ್ಟತೆಯನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
2. **ಫಾರ್ಮ್ಯಾಟಿಂಗ್:** Markdown ಫಾರ್ಮ್ಯಾಟಿಂಗ್ ಸರಿಯಾಗಿ ರೆಂಡರ್ ಆಗುವುದನ್ನು ಪರಿಶೀಲಿಸಿ
3. **ಲಿಂಕ್‌ಗಳು:** ಎಲ್ಲಾ ಆಂತರಿಕ ಮತ್ತು ಬಾಹ್ಯ ಲಿಂಕ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ
4. **ಚಿತ್ರಗಳು:** ಎಲ್ಲಾ ಚಿತ್ರಗಳು ಲೋಡ್ ಆಗುತ್ತವೆ ಮತ್ತು ವಿವರಣಾತ್ಮಕ alt ಪಠ್ಯವನ್ನು ಹೊಂದಿವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
5. **ಸಮತೋಲನ:** ಹಾಲಿ ವಿಷಯದ ರಚನೆ ಮತ್ತು ಶೈಲಿಯನ್ನು ಅನುಸರಿಸಿ

- ಲಿಂಕ್ ಪರಿಶೀಲನೆ

1. ಕನಿಷ್ಠ ಒಂದು ನಿರ್ವಹಕರ ವಿಮರ್ಶೆ ಅಗತ್ಯವಿದೆ
2. ಸೈಬರ್‌ಸುರಕ್ಷತಾ ತತ್ವಗಳ ತಾಂತ್ರಿಕ ಶುದ್ಧತೆಯ ಮೇಲೆ ಕೇಂದ್ರೀಕರಿಸಿ
3. ಪ್ರಾಪ್ತತೆ ಮತ್ತು ಪ್ರಾರಂಭಿಕ ಮಟ್ಟದ ಸ್ನೇಹತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ
4. ವಿಕ್ರೇತ ತಟಸ್ಥತೆಯನ್ನು ಕಾಪಾಡುವುದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

## ಪರೀಕ್ಷೆ ಮತ್ತು ಮಾನ್ಯತೆ

### ವಿಷಯ ಮಾನ್ಯತೆ

ಈ ಡಾಕ್ಯುಮೆಂಟ್ ಯೋಜನೆಯಾದ್ದರಿಂದ, ಪರೀಕ್ಷೆ ಈ ಕೆಳಗಿನವುಗಳ ಮೇಲೆ ಕೇಂದ್ರೀಕೃತವಾಗಿದೆ:

1. **Markdown ರೆಂಡರಿಂಗ್:** Docsify ಬಳಸಿ ಬದಲಾವಣೆಗಳನ್ನು ಸ್ಥಳೀಯವಾಗಿ ವೀಕ್ಷಿಸಿ
2. **ಲಿಂಕ್ ಮಾನ್ಯತೆ:** ಎಲ್ಲಾ ಲಿಂಕ್‌ಗಳನ್ನು ಕೈಯಿಂದ ಕ್ಲಿಕ್ ಮಾಡಿ
3. **ಚಿತ್ರ ಲೋಡಿಂಗ್:** ಎಲ್ಲಾ ಚಿತ್ರಗಳು ಸರಿಯಾಗಿ ಪ್ರದರ್ಶಿಸುತ್ತವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
4. **ಅನುವಾದ:** ಫೈಲ್‌ಗಳು ಅನುವಾದಕ ಕಾರ್ಯಪ್ರವೃತ್ತಿಯಿಂದ ತೆಗೆದುಕೊಳ್ಳುತ್ತವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ (ಸ್ವಯಂಚಾಲಿತ)
5. **ಪ್ರಾಪ್ತತೆ:** ಸರಿಯಾದ ಶೀರ್ಷಿಕೆ ಶ್ರೇಣಿಯನ್ನು ಮತ್ತು alt ಪಠ್ಯವನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

- ಅನುವಾದ ಕಾರ್ಯಪ್ರವೃತ್ತಿ ರಹಸ್ಯಗಳು ಸಂರಚಿಸಲಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿ

- ಬ್ರೌಸರ್ ಕಾನ್ಸೋಲ್‌ನಲ್ಲಿ JavaScript ದೋಷಗಳನ್ನು ಪರಿಶೀಲಿಸಿ

### ಲಿಂಕ್‌ಗಳು ಕೆಲಸ ಮಾಡುತ್ತಿಲ್ಲ

- ಪಾಠಗಳ ನಡುವೆ ಉಲ್ಲೇಖಗಳಿಗಾಗಿ ಸಂಪೂರ್ಣ GitHub URL‌ಗಳನ್ನು ಬಳಸಿ
- ಸ್ವರೂಪ: `https://github.com/microsoft/Security-101/blob/main/[file].md`
- ರೆಂಡರ್ ಮಾಡಿದ ವೀಕ್ಷಣೆಯಲ್ಲಿ ಲಿಂಕ್‌ಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ, ಕೇವಲ ಕಚ್ಚಾ Markdown ನಲ್ಲಿ ಅಲ್ಲ

## ಯೋಜನೆ ನಿರ್ವಹಣೆ

### ನಿಯಮಿತ ಕಾರ್ಯಗಳು

- ಸಮುದಾಯ ಕೊಡುಗೆಗಳನ್ನು ವಿಮರ್ಶಿಸಿ ಮತ್ತು ಮರ್ಜ್ ಮಾಡಿ
- ಸುರಕ್ಷತಾ ಪರಿಸರವು ಅಭಿವೃದ್ಧಿಯಾಗುವಂತೆ ವಿಷಯವನ್ನು ನಿಖರತೆಯಿಗಾಗಿ ನವೀಕರಿಸಿ
- ಅನುವಾದ ಕಾರ್ಯಪ್ರವೃತ್ತಿಯನ್ನು ದೋಷಗಳಿಗಾಗಿ ನಿಗಾ ಮಾಡಿ
- ಸಮಸ್ಯೆಗಳಿಗೆ ಮತ್ತು ಚರ್ಚೆಗಳಿಗೆ ಪ್ರತಿಕ್ರಿಯಿಸಿ
- ಬಾಹ್ಯ ಲಿಂಕ್‌ಗಳನ್ನು ನವೀಕರಿಸಿ

### ಆವೃತ್ತಿ ನಿಯಂತ್ರಣ

- `main` ಶಾಖೆ ರಕ್ಷಿಸಲಾಗಿದೆ ಮತ್ತು PR ವಿಮರ್ಶೆಗಳನ್ನು ಅಗತ್ಯವಿದೆ
- ಎಲ್ಲಾ ಬದಲಾವಣೆಗಳು ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಪ್ರಕ್ರಿಯೆ ಮೂಲಕ ಹೋಗುತ್ತವೆ
- ಅನುವಾದ ಫೈಲ್‌ಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸಲಾಗುತ್ತವೆ, ಕೈಯಿಂದ ಸಂಪಾದಿಸಬೇಡಿ
- ಅರ್ಥಪೂರ್ಣ ಕಮಿಟ್ ಸಂದೇಶಗಳನ್ನು ಬಳಸಿ

## AI ಕೋಡಿಂಗ್ ಏಜೆಂಟ್‌ಗಳಿಗೆ ಟಿಪ್ಪಣಿಗಳು

- **ಇದು ಡಾಕ್ಯುಮೆಂಟ್ ಯೋಜನೆ** - ಕೋಡ್ ಗುಣಮಟ್ಟಕ್ಕಿಂತ ವಿಷಯ ಗುಣಮಟ್ಟದ ಮೇಲೆ ಕೇಂದ್ರೀಕರಿಸಿ
- **ಅನುವಾದ ಫೈಲ್‌ಗಳನ್ನು ಸಂಪಾದಿಸಬೇಡಿ** - ಅವು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರಚಿಸಲಾಗುತ್ತವೆ
- **ಫೈಲ್ ಹೆಸರಿಸುವ ಸಂಪ್ರದಾಯವನ್ನು ಕಾಪಾಡಿ** - ಹಾಲಿ ಮಾದರಿಯಂತೆ ಫೈಲ್‌ ಹೆಸರಿನಲ್ಲಿ ಸ್ಥಳವನ್ನು ಬಳಸಿ
- **README.md ಅನ್ನು ನವೀಕರಿಸಿ** ಮಾಡ್ಯೂಲ್‌ಗಳನ್ನು ಸೇರಿಸುವ/ತೆಗೆದುಹಾಕುವಾಗ ಟೇಬಲ್ ಅನ್ನು ಸಮತೋಲನಗೊಳಿಸಲು
- **PR ಸಲ್ಲಿಸುವ ಮೊದಲು ಸ್ಥಳೀಯವಾಗಿ ಪರೀಕ್ಷಿಸಿ** Markdown ಸರಿಯಾಗಿ ರೆಂಡರ್ ಆಗುವುದನ್ನು ಖಚಿತಪಡಿಸಲು
- **ಹಾಲಿ ವಿಷಯ ಶೈಲಿಯನ್ನು ಅನುಸರಿಸಿ** - ಪ್ರಾರಂಭಿಕ ಮಟ್ಟದ ಸ್ನೇಹತೆಯನ್ನು ಮತ್ತು ವಿಕ್ರೇತ-ತಟಸ್ಥ ಶೈಲಿಯನ್ನು ಕಾಪಾಡಿ
- **ಯಾವುದೇ ಬಿಲ್ಡ್ ಪ್ರಕ್ರಿಯೆ ಅಗತ್ಯವಿಲ್ಲ** - ಸ್ಥಳೀಯ ಅಭಿವೃದ್ಧಿಗಾಗಿ ಸರಳ HTTP ಸರ್ವರ್ ಸಾಕಷ್ಟು
- **ಮಾಡ್ಯೂಲ್ ರಚನೆಯನ್ನು ಗೌರವಿಸಿ** - ಪ್ರತಿ ಮಾಡ್ಯೂಲ್ ಹೊಂದಾಣಿಕೆಯ ಸಂಖ್ಯೀಕರಣ ಮತ್ತು ಸಂಘಟನೆಯನ್ನು ಹೊಂದಿದೆ
